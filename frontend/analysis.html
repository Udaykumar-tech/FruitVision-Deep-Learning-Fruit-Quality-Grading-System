What this shows

Three different models, three different sweet spots. EffNet+SwinT wins on raw accuracy. DBA-ViNet is the smallest. MobileNetV4 dominates interpretability while keeping accuracy strong.

Why no single winner

Different metrics favor different deployment scenarios. A cloud server cares about accuracy. An auditor cares about focus lift. A mobile app cares about size + speed. We trained for all three.

▶

02

Training Curves

How each model learned across 30 epochs. We track validation accuracy, validation loss, cross-entropy loss, and the Path W attention loss (the soft mask-supervision penalty).

What good training looks like

  • • Validation accuracy rising smoothly
  • • Validation loss falling (not rising — that signals overfitting)
  • • Training CE loss eventually plateaus (model has converged)
  • • Path W attention loss drops early then stabilizes (model learned where to look)

What the curves reveal

  • • All 3 models converged cleanly — no divergence, no instability
  • • EffNet+SwinT plateaus highest, fastest — biggest model
  • • DBA-ViNet keeps improving longer — small model, harder fight
  • • Path W loss drops sharply after warmup epoch 3 (when supervision activates)
▶
No model overfit. Training and validation curves stay tightly coupled across all 30 epochs, showing the AggressiveAug pipeline + group-aware splits successfully prevented memorization.
03

Accuracy vs Interpretability

The Pareto frontier visualization. The horizontal axis shows Focus Lift (how much more attention falls on the fruit vs random), the vertical axis shows test accuracy, and bubble size shows parameter count.

How to read this chart

Each bubble is a model. Position on the chart shows the tradeoff:

  • • Top-right = ideal (accurate AND interpretable)
  • • Top-left = accurate but black-box
  • • Bottom-right = interpretable but inaccurate
  • • Larger bubbles = more parameters (heavier to deploy)

Why the >1.8× threshold matters

In XAI literature, a focus lift above 1.8× is the threshold where a model is considered to be "genuinely looking at the object" rather than relying on background shortcuts. The dashed green line marks this. MobileNetV4 sits well past it.

▶
MobileNetV4 dominates the Pareto frontier — sitting in the top-right, proving you don't have to trade accuracy for interpretability if the architecture is well-designed.

Confusion Matrices

Side-by-side confusion matrices for all three models on the test set. Rows = true class, columns = predicted class. The diagonal shows correct predictions; off-diagonal cells reveal specific failure patterns.

How to read confusion matrices

  • • Diagonal cells = correct predictions (deeper blue = better)
  • • Off-diagonal cells = errors (the brighter, the more frequent)
  • • Look for Fresh ↔ Formalin confusion — that's the chemical-detection challenge
  • • Cross-fruit errors (e.g. Apple→Banana) indicate fundamental visual confusion

What the patterns reveal

  • • Almost all errors fall on the Fresh ↔ Formalin axis within each fruit
  • • No model confuses Apple with Banana — fruit identification is solved
  • • DBA-ViNet has the most spread errors, but they're concentrated on the right boundary
  • • Rotten classes have nearly zero errors — visually distinct decay patterns are easy
▶
The hardest task isn't telling apart different fruits — it's distinguishing chemically-treated fruits from genuinely fresh ones. This is the central challenge of the project.
05

Per-Class Recall Heatmap

Recall per class, per model, in a single heatmap. Each cell shows what fraction of that true class was correctly identified.

Reading the heatmap

  • • Green cells = strong recall (close to 1.0)
  • • Yellow cells = mediocre (around 0.7–0.85)
  • • Red cells = weak recall (below 0.7)
  • • Look across rows to compare classes; across columns to compare models

Practical interpretation

A class with low recall means the model often misses that class when it's truly present — bad for safety-critical detection. Formalin classes are the typical weak points. EffNet+SwinT achieves uniformly high recall, while DBA-ViNet sacrifices some recall for its smaller size.

▶
Rotten and Fresh classes consistently sit above 0.93 recall across all models. The Formalin row is where models are stress-tested — and where MobileNetV4 punches above its weight.
06

Formalin Detection — The Hardest Task

Per-fruit recall and precision for the Formalin-Mixed class only. This is what the project is fundamentally about: detecting fruits chemically treated with formalin, even when they visually appear fresh.

Why formalin detection is hard

Formalin preservation masks the natural decay process. A formalin-treated fruit often looks shinier and "more perfect" than a regular fresh fruit. The signal is in subtle skin-pore patterns and unnatural sheen — easily missed by models that focus on background or shape.

Reading the bars

  • • Recall = of all formalin fruits, how many we caught
  • • Precision = of all "formalin" predictions, how many were right
  • • Both matter: missing formalin = food safety risk; false alarm = wasted produce
  • • The dashed red line marks 0.85 — a typical "deployable" threshold
▶
EffNet+SwinT achieves > 0.95 recall on every formalin class — a strong result. MobileNetV4 trails by a small margin (~0.02–0.04) while being half the size, making it the most practical choice for actual deployment.
07

Focus Score Distributions

For each test image, we measure what fraction of the Grad-CAM heatmap energy lands inside the actual fruit region (vs the background). Higher = the model genuinely looks at the fruit.

What "focus score" means

We use SAM-generated masks of the fruit region, then check how much of the model's attention falls inside that mask:

  • • 0.0 = model attends entirely to background
  • • 0.19 = random baseline (roughly the fraction of pixels inside fruit)
  • • 1.0 = model attends only to fruit, never background

What the distributions reveal

  • • EffNet+SwinT histogram peaks near baseline — uses background features
  • • DBA-ViNet shifts right substantially — architectural attention works
  • • MobileNetV4 mass is heavily right-skewed — mostly looks at fruit
  • • Box plots show median + spread — confirms the same trend
▶
This is the smoking-gun XAI evidence. The shift from EffNet (0.27 mean focus) to MobileNetV4 (0.86 mean focus) represents a 3× improvement in where the model actually looks when making decisions.
08

Grad-CAM Visual Comparison

The visual proof. For one sample image per class (15 rows), we show the original alongside each model's Grad-CAM overlay. Where each model's heatmap sits tells you what it considered important for that prediction.

How Grad-CAM works

Grad-CAM computes which spatial regions in the input image most increased the score of the predicted class. Red = strong influence on the decision. Blue/cool colors = ignored areas. We use late-conv-layer Grad-CAM for all models.

What to look for in each row

  • • EffNet+SwinT often has heatmaps spilling onto background
  • • DBA-ViNet centers on fruit but sometimes too narrow
  • • MobileNetV4 consistently lights up fruit skin texture
  • • For Rotten classes, look for hot spots on the decay region itself
▶
A picture is worth a thousand metrics. Scrolling through this grid is the single most convincing demonstration that interpretability is real — not just a number we report.
09

Inference Speed

Wall-clock latency per image (256×256, AMP, single image). Measured on the development hardware: NVIDIA RTX 3070 Laptop with 8 GB VRAM.

Why latency matters

  • • Real-time applications need < 100ms (10+ FPS)
  • • Mobile devices typically run 5–15× slower than desktop GPUs
  • • Batch processing benefits from larger throughput, not lower latency
  • • Edge accelerators (Edge TPU, Apple Neural Engine) further amplify these gaps

Reading the bars

  • • EffNet+SwinT heaviest — two backbones running in parallel
  • • DBA-ViNet middle — single backbone + small transformer
  • • MobileNetV4 fastest — UIB + Mobile MQA optimized for hardware
▶
MobileNetV4 isn't just the most interpretable — it's also the fastest. This is the trifecta required for actual deployment: accurate, interpretable, and fast.
